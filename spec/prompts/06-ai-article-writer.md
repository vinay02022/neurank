# Phase 06 — AI Article Writer (Instant / 4-Step / 10-Step) + Brand Voice

**Goal:** The flagship content creation pipeline that generates long-form, SEO-optimized, fact-checked articles with citations, FAQs, internal links, and a cover image.

Spec: `prd.md` §7.6 and §7.7.

## 1. Brand Voice

### 1.1 Create Brand Voice
`src/app/(app)/content/brand-voices/new/page.tsx`

Form with three training methods (tabs):
- Paste text (min 300 words)
- Upload files (PDF/DOCX up to 10 MB) — parse via `pdf-parse` and `mammoth`
- Add URLs (max 5) — fetch + extract readable text via `@mozilla/readability` + `jsdom`

On submit:
1. Combine samples into a single `sampleText`.
2. Call `router.generate` with `task: "brand-voice:extract"` and schema:

```ts
z.object({
  tone: z.array(z.enum(["formal","casual","witty","authoritative","playful","expert","empathetic"])),
  pointOfView: z.enum(["first_person_singular","first_person_plural","second_person","third_person"]),
  avgSentenceLength: z.number(),
  signaturePhrases: z.array(z.string()).max(20),
  vocabularyLevel: z.enum(["simple","standard","advanced"]),
  dosAndDonts: z.object({ dos: z.array(z.string()), donts: z.array(z.string()) })
})
```

Save result as `profileJson` on `BrandVoice`.

## 3. Wizard forms

Use a multi-step form with zod schemas per step. Persist intermediate state in URL (`?step=2`) and draft in DB (`Article` with `status: DRAFT`).

### 3.1 Instant (`/content/articles/new?mode=instant`)
Single screen:
- Topic/Title (text)
- Article type (select: listicle, how-to, news, comparison, definition, case study, review)
- Language + Country
- Optional keywords (tag input)
- "Generate" button → debits 20 credits → triggers `article/generate.requested`

### 3.2 4-Step (`?mode=step4`)
Step 1: type + keywords  
Step 2: title  
Step 3: outline (generated by LLM, user can edit headings + reorder with `dnd-kit`)  
Step 4: review & generate

### 3.3 10-Step (`?mode=step10`)
Same as 4-Step plus:
- Reference / competitor URLs (we fetch + summarize)
- Word length (slider 500-5000)
- Brand voice selector
- CTA text + URL
- Cover image options (generate with DALL·E 3 vs upload)
- Internal link targets (auto-suggest from existing articles)
- FAQ questions (auto-generated, editable)
- Advanced: include stats / examples / quotes toggles

## 4. Generation pipeline — `src/server/inngest/article-generate.ts`

Inngest function driven by `article/generate.requested` event.

Steps (each `step.run`):

1. **Research** — Tavily or Serper + top 5 results; scrape top 3 with Readability; summarize each with `task: "article:outline"`.
2. **Outline** — `generateObject` returning:
   ```ts
   z.object({
     h1: z.string(),
     sections: z.array(z.object({
       heading: z.string(),
       subheadings: z.array(z.string()).optional(),
       keyPoints: z.array(z.string()),
       targetWords: z.number()
     }))
   })
   ```
3. **Sections** — one LLM call per section, streamed to DB (append to `contentMd`). Prompt includes:
   - Global style guide (Brand Voice `profileJson`)
   - Research summaries
   - SEO hints (keywords, LSI terms via Serper "related searches")
   - Internal link anchors (pass a list of {url, anchor} — let model use where fitting)
4. **Fact-check** — second LLM pass flags claims lacking citations; insert `[[cite: url]]` markers for claims tied to our research snippets.
5. **FAQ** — generate 5-7 Q&A pairs as JSON; store in `faqJson`; render as JSON-LD on publish.
6. **Cover image** — if requested, `ai.router` → OpenAI `gpt-image-1` (or DALL·E 3) with prompt derived from title; upload to Vercel Blob / S3 (use `@vercel/blob`); save URL on Article.
7. **Compile** — assemble `contentMd` + `contentHtml` (via `remark` + `rehype`), calculate final word count, set `status: GENERATED`.
8. **Telemetry** — insert `LLMEvent` rows already captured by router; update article's `creditsSpent`.

If any step fails: set `status: FAILED`, record error on an `ArticleEvent` log (add minimal table or reuse AuditLog), send toast to user.

## 5. Editor — `src/app/(app)/content/articles/[id]/page.tsx`

- Left: article content in a live Markdown editor (use `tiptap` with a Markdown extension, or `react-markdown-editor-lite`, or plain textarea with split preview for MVP)
- Right: metadata panel — word count, SEO score (quick client-side heuristic), keywords density, brand voice, FAQs
- Top actions: Save, Regenerate section, Export (MD/HTML/DOCX via `docx`), Publish to WordPress

## 6. Publish to WordPress

Connect WordPress credentials via settings → integrations (store Application Password in env-encrypted workspace settings). "Publish" server action:
1. `POST https://{site}/wp-json/wp/v2/posts` with HTML + title + categories + tags
2. On success: set `publishedUrl` on Article + AuditLog.

For MVP, fallback: "Copy HTML" button works even without WP.

## 7. Public API — `/api/v1/articles/instant`

POST with API key (from phase 01 `ApiKey` table; compare `hashedKey` using bcrypt):
```json
{ "topic": "...", "type": "listicle", "language": "en" }
```
Rate-limited via Upstash. Returns `{ articleId, status: "GENERATING" }` immediately; the consumer polls `GET /api/v1/articles/:id` for the final content.
